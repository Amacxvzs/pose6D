# Project Structure

## Keep

- `configs/`: capture, calibration, object, and YOLO configuration.
- `data/raw/`: original RGB-D captures and metadata.
- `data/labels/`: manual labels and reviewed-label archive.
- `data/yolo/`: original YOLO dataset used by the baseline model.
- `data/yolo_pseudo_20260617/`: current expanded YOLO dataset.
- `models/plate/`: plate CAD model and AABB geometry.
- `runs/detect/plate_yolov9t/`: baseline training result.
- `runs/detect/plate_yolov9t_pseudo_20260617/`: current training result.
- `runs/detect/compare_test_*_corrected_20260618/`: corrected test comparison.
- `scripts/`: reusable capture, training, pose, analysis, and thesis scripts.

## Current Outputs

- `outputs/detections_pose/`: compact detector CSV/JSON output.
- `outputs/pose_quality/`: pose quality classification.
- `outputs/occlusion_plan/`: stage-one/stage-two processing plan.
- `outputs/two_stage_poses/`: merged two-stage poses before table constraint.
- `outputs/two_stage_poses_table_constrained/`: final table-constrained poses.
- `outputs/two_stage_review_table_constrained/`: final visual review sheets.
- `outputs/failure_review/`: skipped/failure cases.
- `outputs/thesis_figures/`: figures for the thesis.
- `outputs/ablation_experiment/`: ablation summaries and charts.

## Regenerated Intermediates

The following directories can be regenerated by the pipeline and do not need
manual backup:

- `outputs/object_clouds/`
- `outputs/initial_poses/`
- `outputs/icp_poses/`
- `outputs/object_clouds_stage2/`
- `outputs/initial_poses_stage2/`
- `outputs/icp_poses_stage2/`

Run `scripts/run_detector_plate.bat`, the three first-stage pose scripts, and
`scripts/run_two_stage_pose_pipeline.bat` to rebuild them.
